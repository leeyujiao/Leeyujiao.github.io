第二章：Java内存区域与内存溢出异常

运行时数据区域：

可按照线程的私有公有分类：
线程私有：
程序计数器，Java虚拟机栈，本地方法栈；
线程公有：
Java堆，方法区（非堆）；
其中，除了程序计数器以外，其他区域都可以抛出OutOfMemoryError异常（我的理解是内存不够用的时候就会抛出此异常），除了这个异常还有一个StackOverflowError异常（这是一个跟栈有关的异常，线程要求的栈深度大于虚拟机允许，就会抛出此异常，所以它只会被Java虚拟机栈和本地方法栈抛出）；

程序计数器：
如果线程执行Java方法，程序计数器存储指令地址；但如果执行Native方法，计数器就为空。（Native方法就是计算机函数，它的实现由非Java语言完成）
Java虚拟机栈：
存储局部表量表，操作数栈，动态链接，方法出口等信息；
局部变量表存放编译器可知的基本数据类型，引用类型，returnAddress类型（指向一条字节码指令地址）；局部变量表在编译期间就完成了分配，方法运行期间它的大小不会发生改变；
本地方法栈：
类似于Java虚拟机栈，只不过，Java方法在Java虚拟机栈里，Native方法在本地方法栈中；
Java堆：
虚拟机所管理内存最大的一块，用来给对象实例和数组分配内存，它是垃圾收集器管理的主要区域，被称为GC堆（莫名觉得这个名字略有趣 Garbage Collected Heap）。
方法区：
用于储存被加载过的类信息，常量，静态变量，即时编译器编译后的代码等，除了可选择固定大小，可扩展且不需连续内存，还可以不实现垃圾收集，所以易因未完全回收出现内存泄漏。

Ajax

发表于 2019-10-20 分类于 Ajax与JSON

Ajax:
Asynchronous Javascript And xml 异步的JavaScript和XML。
1，异步和同步：客户端和服务器端相互通信的基础上
客户端必须等待服务器端的响应，在等待期间不能做其他操作；
客户端不需要等待服务器端的响应，在服务器处理请求的过程中，客户端可以进行其他操作

AJAX是一种在无需重新加载整个页面的情况下，能够更新部分网页的技术。
通过在后台与服务器进行少量数据交换，Ajax可以使网页实现异步更新，这意味着可以在不重新加载整个页面的情况下，对网页的某部分进行更新。
传统的网页（不使用Ajax）如果需要更新内容，必须重载整个网页页面
提升用户体验；

实现方式：
1.原生的js实现方式；

function ajaxdemo(){
    //第一步 创建Ajax对象
    var ajax=new XMLHttpRequest();
    //第四步：回调
    ajax.onreadystatechange=function () {  //当有回调时
        var ststus1=ajax.status;  //状态码
        var readyStatel=ajax.readyState;//
        if (ststus1==200&&readyStatel==4) {//200 路径正确，servlet未出错  500客户端错    400    服务器错误     4 全完成    3 未返回回应 2 未send  1  创建未open 0 未创建
            var result = ajax.responseText;   //返回结果     做相应
            alert(result);
        }
    }
    //第二步 打开链接通道
    ajax.open("get","a?a=3&b=4",true);  //第三个参数是   是否异步
    //三：发送
    ajax.send();
}

2.Jquery实现方式；
$.ajax():
语法：$.ajax({键值对});

//定义方法
function fun() {
    //使用$.ajax()发送异步请求
    $.ajax({
       url:"ajaxServlet",//请求路径
        type:"post",  //请求方式 默认get
       // data:"username=jack&age=23",          //携带的参数方法一
        data:{"username":"jack","age":23},//方法二 推荐
        //回调函数
        success:function (data) {  //data接受服务器返回结果的值
            alert(data)
        },//响应成功后的回调函数
        error:function () {
            alert("出错了")
        },//表示如果请求响应出错，会执行的回调函数
        dataType:"text"//设置受到的响应的数据格式

        });

$.get():发送get请求
语法：$.get(url,[data],[callback],[type])
参数：
url:请求路径
data:请求参数
callback:回调函数
type:响应结果的类型

//定义方法
function fun() {
    $.get("ajaxServlet",{username:"tom",age:23}, function (data) {
        alert(data);
    },"text");
}

$.post():发送post请求

//定义方法
function fun() {
    $.post("ajaxServlet",{username:"tom",age:23}, function (data) {
        alert(data);
    },"text");
}

MyBatis学习01

发表于 2019-10-15 更新于 2019-10-20 分类于 MyBatis

MyBatis项目的创建流程：

1.利用maven创建quickstart项目并导包（sql包和mybatis包）；
2.在main文件夹下创建一个包，右击mark为Resources型，在该包中创建一个名为config.xml的文件，在其中配置环境；
3.在Java文件夹下创建bean包，创建一个Info类来写被操作表中的属性和set，get，toString方法；
4.在Java文件夹下创建mapper包，再创建mapper接口和.xml文件，接口中定义函数，xml中写sql语句并绑定函数；
5.建一个test类产生对象调用方法；

关于sql语句中的#和$

符号#类似于通配符，和sql语句结合；
符号$是直接补全sql语句，若有存在字符型，字符串型就需要在字符/字符串上加‘’，否则sql语句不能执行；

需多参数传递的解决方法

Mapper接口中的方法不能传递多参数，故需要封装：
1.可以封装成为Info类型；
2.可以封装成集合（好处是可以自定义属性名称）；

列名和属性名不匹配

1.在sql语句中直接用 ...as...改正；
2.在Mapper.xml中加入<resultMap>标签，子标签<id>和<reult>用来定义转换关系，主键用<id>，非主键用<reult>；
